Expresion regular

¿Que es? Importancia Casos de uso

¿Que es?

Una expresión regular es una secuencia de caracteres que define un patrón de búsqueda. Estas expresiones regulares se utilizan para describir secuencias específicas de caracteres en documentos o cadenas de texto. A menudo, su objetivo es modificar, buscar o reemplazar fragmentos de texto o líneas de código.

Importancia

Las expresiones regulares son una herramienta muy poderosa para buscar y manipular texto. Son una característica común en muchos lenguajes de programación y herramientas de procesamiento de texto, y proporcionan una forma poderosa y flexible de buscar y manipular texto. Las expresiones regulares son especialmente útiles para buscar patrones de texto en grandes cantidades de texto, como archivos de registro o archivos de configuración.

Expresiones regulares más utilizadas

  • Blackslash o barra invertida: \
    Se utiliza para escapar el siguiente carácter de la expresión de búsqueda de forma que este adquiera un significado especial o deje de tenerlo. Al utilizarlo por ejemplo en combinación con el punto “\.” este deja de tener su significado normal y se comporta como un carácter literal.
  • Pipe o barra: |
    Sirve para indicar una de varias opciones. Por ejemplo la expresión regular (indexingdata|seo|sem) permitirá encontrar cualquiera de esas palabras.
  • Punto: .
    El punto se interpreta como cualquier carácter, es decir, busca cualquier carácter o incluso un espacio SIN incluir los saltos de línea.
  • Asterisco: *
    Sirve para encontrar algo que se encuentra repetido 0 o más veces. Su utilización más frecuente es junto al punto .* para indicar “cualquier cadena alfanumérica”.
  • Circunflejo: ^
    Permite indicar cuál es el inicio de una cadena de texto.
  • Dólar: $
    Permite indicar cuál es el final de una cadena de texto.
  • Paréntesis: ( )
    Permiten agrupar una consulta, de manera que podamos construir partes de una RegEx mayor, o garantizar que no se mezclan las cadenas.
  • Corchetes: [ ]
    Permiten indicar un intervalo de números o caracteres, de manera que cualquiera incluido en el intervalo valide la consulta.
  • Interrogación: ?
    Indica que el carácter o cadena de delante del signo es opcional.
  • [abc] : Uno de los caracteres” a”, “b”, o “c”
  • [^abc] : Cualquier carácter menos: “a”, “b”, o “c”
  • [a-z] : Cualquier minúscula entre la ”a” y la “z”
  • [0-9] : Cualquier número entre el “0” y el “9”
  • [a-zA-Z] : Cualquier mayúscula o minúscula entre la “aA” y la “zZ”
  • [a-zA-Z0-9-] : Cualquier letra o número o el guión (ideal para URLs)
  • .* : Lo de antes puede existir ninguna vez o más
  • .+ : Lo de antes puede existir una vez o más

Casos de uso

Las expresiones regulares se utilizan en muchos campos, como la programación, la minería de datos, la extracción de información, la búsqueda de texto, la validación de datos, la manipulación de texto y la sustitución de texto. Algunos ejemplos de casos de uso comunes incluyen:

  • Validar un nombre de usuario.
  • Validar una dirección de correo electrónico.
  • Validar un número telefónico.
  • Encontrar urls en un texto.
  • Validar si un texto contiene caracteres especiales.
  • Validar las características de una contraseña, por ejemplo, que contenga números, mayúsculas, minusculas y caracteres especiales.
  • Encontrar palabras que empiecen con una letra en específico.
  • Validar una dirección IP.
  • Validar una fecha.
  • Encontrar números decimales en un texto.
  • Validar un código postal.
  • Validar un RFC.
